00006 @section Description
00007 
00008 The main objective of the MultiFit framework is to provide a common fitting interface to different
00009 parametric models and fitting algorithms.
00010 
00011 To this end, MultiFit provides a common programming interface (API) to different minimizers and 
00012 parametric models. 
00013 This allows in particular to:
00014 - Test how well different minimizers perform when fitting the same model. This is useful, 
00015   for instance to evaluate different implementations of the Levenberg-Marquardt non-linear least 
00016   squares algorithm (LVM) (e.g. levmar, http://users.ics.forth.gr/~lourakis/levmar/ or 
00017   scipy.leastsq, http://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.leastsq.html). 
00018 - Evaluate minimization algorithms other than LVM such as Coordinate Descent (CD), 
00019   http://en.wikipedia.org/wiki/Coordinate_descent.  
00020 - Keeping the same minimizer and the same data, select the model that give the most accurate 
00021   results. In the context of astronomy, one can for instance compare models based on 
00022   single-component Sersics and Bulge+Disk Sersics.
00023 
00024 This version is still in beta. Only the following minimizers and astronomical models are supported:
00025 - Minimizers:
00026   - @c scileastsq: based on the scipy Levenberg-Marquardt non-linear least available with the 
00027                 Python scipy library 
00028                 (http://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.leastsq.html))
00029   - @c scdmin: an implementation of the Coordinate Descent algorithm, based on 
00030                SCDM (Simple Coordinate Descent Minimizer, Marc Gentile, 2013).
00031 
00032 - Parametric models:
00033   - @c scsersic: an implementation of galaxy model base on a pure disk Sersic model 
00034               (http://en.wikipedia.org/wiki/Sersic_profile)  
00035   - @c gbdsersic: an implementation of galaxy model based on the combination of an exponential Sersic
00036                profile (sersic index 1) to model the galaxy disk and a de Vaucouleur Sersic profile
00037                (Sersic index 4) to model the galaxy bulge. Both the disk and the bulge share the
00038                same centroid and the same elipticity. The underlying implementation is done using
00039                @c GalSim v1.x, the "modular galaxy image simulation toolkit", 
00040                https://github.com/GalSim-developers/GalSim.
00041   - @c gcbdsersic: an implementation of galaxy model identical to that of @c gbdsersic except the
00042                 disk Sersic index can also vary. 
00043   - @c profile2D: model of a generic two-dimensional profile, mostly for testing purpose.
00044 
00045 The software is designed so that additional models and fitting methods can be addded fairly quickly.
00046
