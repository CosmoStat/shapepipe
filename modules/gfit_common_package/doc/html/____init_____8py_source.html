6 @section Description
7 
8 The @c gfit method uses a forward model fitting algorithm to measure galaxy shapes.
9 
10 An earlier version of gfit, used to participate in the @c GREAT10 galaxy Challenge
11 (Kitching et al., 2011, Applied Statistics, Kitching et al, 2012, MNRAS), has been described
12 in Gentile et al, 2012, astro-ph/1211.4847 (http://arxiv.org/abs/1211.4847).
13 
14 The present version has been customized to participate in the @c GREAT3 Challenge
15 (http://great3.projects.phys.ucl.ac.uk/leaderboard/).
16 
17 The following galaxy models are currently supported:
18  - @c scsersic: an implementation of galaxy model base on a pure disk Sérsic model
19  (http://en.wikipedia.org/wiki/Sersic_profile)
20  - @c gbdsersic: an implementation of galaxy model based on the combination of an exponential
21  Sérsic profile (sersic index 1) to model the galaxy disk and a de Vaucouleur Sérsic
22  profile (Sérsic index 4) to model the galaxy bulge. Both the disk and the bulge
23  share the same centroid and the same elipticity. The shear estimator is the
24  distortion |e| = (a^2 - b^2)/(a^2 + b^2). The underlying implementation is
25  done using @c GalSim v1.x, the "modular galaxy image simulation toolkit", see
26  https://github.com/GalSim-developers/GalSim.
27  - @c gbdshsersic:
28  - @c gbdshsersic: an implementation of galaxy model identical to that of @c gbdsersic except that
29  the reduced shear |g| = (a - b)/(a + b) is used insead of the distortion |e|.
30  - @c gcbdsersic: an implementation of galaxy model identical to that of @c gbdsersic but with
31  a varying disk Sérsic index.
32  - @c gccbdsersic: an implementation of galaxy model identical to that of gcbdsersic except that
33  the disk and the bulge Sérsic index can also vary.
34 
35 Fitting can currently be performed using two minimizers:
36  - @c scileastsq: based on the <tt>Levenberg-Marquardt</tt> non-linear least-squares
37  implementation, available with the Python SciPy library
38  (http://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.leastsq.html))
39  - @c scdmin: a wrapper to SCDM (Simple Coordinate Descent Minimizer, M. Gentile, 2011-2013)
40  an implementation of the Coordinate Descent algorithm
41  (http://en.wikipedia.org/wiki/Coordinate_descent).
42 
43 @c gfit is written in @c Python (http://www.python.org/) and takes advantage of the language
44 flexibility and richness of features. It relies heavily on the @c numpy and @c scipy
45 @c Python C/C++/fortran libraries.
46 
47 The software has been designed in such a way that additional galaxy models and minimizers can be
48 plugged-in fairly quickly.
49 
50 The implementation of this version relies mainly on the following modules:
51 
52 - ../../mpf_package/mpf (Multi-Processing Framework, M. Gentile, 2013): a Python module for executing code on
53  both SMP (Symmetric Multiprocessing) or on serveurs supporting the Message Passing Interface
54  (MPI). Refer to the documentation of @c mpf module for additional details.
55 - @c mpfg3 (M. Gentile, 2013): an extension of @c mpf that provides an adaptation layer to access
56  the @c GREAT3 datasets. Refer to the documentation of @c mpfg3 module for additional details.
57 - @c multifit (MulTipurpose Fitting framework, M. Gentile, 2013): a python module that offers a
58  common fitting interface to different parametric models and fitting algorithms. Refer to the
59  documentation of @c Multifit for additional details.
60 - @c scdm (Simple Coordinate Descent Minimizer, M. Gentile, 2011-2013) an implementation of the
61  Coordinate Descent algorithm (http://en.wikipedia.org/wiki/Coordinate_descent).
62 - Numpy v1.7.x., SciPy v0.9.x and pyfits Python modules

121 @section Execution
122 
123 The general syntax is:
124 
125 - SMP version: @code gfit_SMP.py [gfit options] @endcode
126 - MPI version: @code gfit_MPI.py [gfit options] @endcode
127 
128 These executables are located in the installation directory under <tt>gfit-0.7.5/gfit</tt>.
129 By default, gfit will look for a configuration file with default name @c gfit.cfg, located in the
130 <tt>./gfit/config directory</tt>. These settings can be changed using the @c -c and
131 @c -d options (see below).
132 
133 The supported @c gfit options are:
134 
135 @verbatim
136  -h, --help Display the usage syntax and the list of supported options.
137 
138  -d, --config-dir Directory where to find the configuration file (default: ./config)
139 
140  -c, --config-file Name of the configuration file (default: gfit.cfg)
141 @endverbatim
142 
143 To run @c gfit_MPI.py, one has to use the mpirun executable as:
144 
145 @code mpirun -np N gfit_MPI.py [gfit options]@endcode or
146 @code mpirun -np N python [python options] gfit_MPI.py [gfit options]@endcode
147 
148 where @c N is the number of processors (or nodes) to use. @c MPI must has been
149 installed and configured appropriately.
150 On a cluster with a queuing system managed by a job scheduler, @c mpirun will have to be integrated
151 in a shell script, invoked using a command like @c qsub.
152 
153 Since the manager process of gfit uses one processor for itself, workers will
154 share <tt>N-1</tt> processors.
155 
156 For example: @code mpirun -np 6 gfit_MPI -d mydir -c myconfig.cfg @endcode will run @c gfit
157 on 6 processors, 1 for the manager and 5 for the workers. The manager and each
158 of the workers will look for a configuration file: <tt>./mydir/myconfig.cfg </tt>.
159 
160 @note Before running @c gfit, edit the configuration file to set the @c BASE_INPUT_DIR value, which
161  should point to the directory where the @c GREAT3 files reside.
162 @note In the configuration file, any environment variable prefixed with @c $ such as @c $HOME will
163  be expanded when read.

165 @section gfit_config Configuration
166 
167 The behaviour of @c gfit can be controlled though various configuration options located in its
168 configuration file, <tt>gfit.cfg</tt> by default, in the <tt>gfit/config</tt> directory.
169 
170 The format of the configuration file is similar to that of MS Windows&trade; .ini text files, with
171 @c sections embedded within brackets, like <tt>[section]</tt>, each section containing a set of
172 (@c key, @c value) pairs. The <tt>.ini</tt> format has been extended to support subsections and
173 section inheritance. Configuration files are managed by the @c sconfig python module, wiich should
174 be separately installed (see section <b>Prerequisites</b> above).

176 @subsection dirs Directories
177 
178 The input, output and corrresponsing sub-directories are specified under the <tt>[DIR]</tt> section.
179 
180 @verbatim
181 # --- This section contains the directories from where gfit will read or store data
182 [DIR]
183 
184 [DIR.INPUT] # base input directories: currently unused
185 BASE_INPUT_DIR = ./input # location of input objects to process
186 
187 [DIR.OUTPUT] # base output directories
188 BASE_OUTPUT_DIR = ./output # where to store results, etc.
189 OUTPUT_PLOT_DIR = plots # where to save plots
190 OUTPUT_MAP_DIR = maps # where to save maps
191 OUTPUT_STAT_DIR = stats # where to record statistics
192 OUTPUT_LOG_DIR = logs # where to write logs
193 OUTPUT_RESULT_DIR = results # where to write results of runs
194 OUTPUT_ERROR_DIR = errors # where to write errors during runs
195 @endverbatim
196 
197 Each time @c gfit is run, a directory of the form:
198 <tt>run_DD.MM.YY_HH.MM.SS.branch.obs_type.data_type</TT> is created under the output directory
199 specified by the key @c BASE_OUTPUT_DIR under the section @c DIR.DIR_OUPUT. The @c branch,
200 @c obs_type and @ data_type components are named after the first capitalized letter of the branch,
201 data type and observation types respectively.
202 
203 For instance a run in the control-space-variable
204 branch will have its data contained in a directory of the form <tt>run_DD.MM.YY_HH.MM.SS.CSV</TT>.
205 Or for a run in both the variable_psf-space constant and the full-ground-variable branches,
206 a directory like <tt>run_DD.MM.YY_HH.MM.SS.VSC_FGV</TT> will be created. The created directory
207 will in turn contains the sub-directories @c plots, @c maps, @c stats @c logs, @c results and
208 @c errors. The @c plots, @c maps, @c stats and @c results will have a directory tree-like structure
209 that mimics the structure of the dataset.
210 
211 The <tt>DIR.INPUT</tt> section is currently unused.

213 @subsection datasets Datasets
214 
215 The @c [DATASET] section specifies where input images, catalogs an other requred files to process
216 are located. It also specifies Unix-like file matching patterns used to identify those files.
217 At a minimum, a @c dataset has a name, a base directory and a a list of file matching patterns.
218 
219 In this version @c gfit uses 3 datasets: one to access the @c GREAT3 images, catalogs and other
220 files, @c SExtractor galaxy and PSF catalog files and a @c PSF dataset with PSF images at the
221 positions of galaxies. These datasets are described in more detail below.
222 
223 @subsubsection g3_dataset GREAT3 dataset
224 
225 The @c GREAT3 dataset is identified by the <tt>DATASET</tt> section:
226 
227 @verbatim
228 # --- Main source dataset, in this case GREAT3
229 [DATASET] # note: cannot process both deep and normal images in the same run
230 
231 NAME = great3 # name of dataset to use
232 BASE_DIR = $HOME/GREAT3 # dataset base input directory
233 
234 # --- gfit needs galaxy images and catalogs
235 FILE_PATTERNS = ["image*.fits", "galaxy_catalog*"]
236 
237 # Note: to process deep images, use: FILE_PATTERNS = ["deep_image*", "deep_galaxy_catalog*"]
238 
239 # --- Search criteria, branches: ["control", "real_galaxy", "variable_psf", "multiepoch", "full"]
240 BRANCHES = ["control"]
241 
242 # --- Observation type: ["ground", "space"]
243 OBS_TYPES = ["ground"]
244 
245 # --- Data type: ["constant", "variable"]
246 DATA_TYPES = ["constant"]
247 
248 # --- Set of Image numbers (use [] to process all images (IMAGE_LIST has priority over IMAGE_RANGE)
249 IMAGE_LIST = [0]
250 
251 # --- [Min,Max] image number range (-1 or [] to disable criterion)
252 IMAGE_RANGE = []
253 
254 # --- Actual Galaxy stamp size to use for shape measurement
255 GALAXY_EFFECTIVE_STAMP_SIZE = {"control":{"ground":26, "space":54},\
256  "multiepoch":{"ground":44, "space":44},\
257  "variable_psf":{"ground":44, "space":88},\
258  "real_galaxy":{"ground":44, "space":88},\
259  "full":{"ground":44, "space":88}\
260  }
261 
262 # --- Actual PSF stamp size to use for shape measurement
263 PSF_EFFECTIVE_STAMP_SIZE = {"control":{"ground":26, "space":54},\
264  "multiepoch":{"ground":44, "space":44},\
265  "variable_psf":{"ground":44, "space":88},\
266  "real_galaxy":{"ground":44, "space":88},\
267  "full":{"ground":44, "space":88}\
268  }
269 @endverbatim
270 
271 The <tt>BASE_DIR</tt> key specifies where the base @c GREAT3 directories are located, i.e. where
272 the branch directories (@c control, @c real_galaxy, @c multiepoch, etc.) are to be read from.
273 
274 The <tt>FILE_PATTERNS</tt> key contains a list of pattern matching strings, used to identify which
275 files are collectively needed to perform shape measurement. A set of such files is processed by
276 @c gfit as a @c job.
277 For instance, in the context of @c GREAT3, a job will contain all files related to image @c 000
278 in the <tt>control-ground-constant</tt> branch tree: <tt>image-000.0.fits</tt> and
279 <tt>galaxy_catalog-000.fits</tt>. On a multiprocessor machine or cluster, each job will be assigned
280 to one processor.
281 
282 The <tt>BRANCHES</tt>, <tt>OBS_TYPE</tt> and <tt>DATA_TYPE</tt> keys respectively specify the lists
283 of branches (e.g. "control"), observation types (e.g. "ground", "space") and data types (e.g.
284 "constant", "variable") to process.
285 
286 The <tt>IMAGE_LIST</tt> key is a list of image numbers to process, like for instance
287 <tt>[0, 10, 67]</tt>.
288 An empty list <tt>[]</tt> indicates that @e all images in the underlying branch, observation type
289 and data type have to be processed.
290 
291 The <tt>IMAGE_RANGE</tt> key specifies a range of image numbers to process. for instance, [1, 50]
292 will have @c gfit access all images from 1 to 50, 1 and 50 included. Note that the
293 <tt>IMAGE_LIST</tt> has priority other <tt>IMAGE_RANGE</tt>: the image range will be ignored unless
294 <tt>IMAGE_RANGE=[]</tt>.
295 
296 The <tt>GALAXY_EFFECTIVE_STAMP_SIZE</tt> and <tt>PSF_EFFECTIVE_STAMP_SIZE</tt> keys respectively
297 control the sizes of the postage stamps to cut-out from galaxy and PSF images. @c gfit will
298 estimate the centroid of the postage stamp image and cut-out a stamp with the specified size.
299 Note that since most centroids are shifted by 1 or more pixels, cutting-out a square stamp of the
300 exact size specified in <tt>GALAXY_EFFECTIVE_STAMP_SIZE</tt> and <tt>PSF_EFFECTIVE_STAMP_SIZE</tt>
301 may not be possible. In such cases, @c gfit will automatically reduce the size of the stamp to the
302 maximum possible size. A warning will be issued in the log file for that image.

410 @subsection shape Shape measurement
411 
412 The <tt>[SHAPE_MEASUREMENT]</tt> controls the shape measurement operation.
413 
414 @verbatim
415 [SHAPE_MEASUREMENT]
416 NORMALIZE_PSF = True # if True, normalize the PSF image before convolution
417 SUBSTRACT_GALAXY_SKY = False # if True estimate and remove the sky noise in galaxy images
418 SUBSTRACT_PSF_SKY = False # if True estimate and remove the sky noise in PSF images
419 FAILED_ELLIPTICITY_VALUE = 10.0 # if -1,.0 take the value of the fit, even though it failed
420 GALAXY__SUBSCALE = 1 # resolution of constructed galaxy models
421 @endverbatim
422 
423 The key <tt>FAILED_ELLIPTICITY_VALUE</tt> sets the value of ellipticity used to flag wrong fits in
424 the results submission files produced for each processed image. The @c GREAT3 presubmission script
425 will ignore entries with ellipticity values greater than or equal to @c 10. If set to @c -1,
426 @c gfit will not flag entries with bad fits. This is not recommended.
427 
428 Note that @c gfit also always produce, for each image, a result submission file that
429 includes @e all fits either good or bad. So the preferred approach is to choose
430 <tt>FAILED_ELLIPTICITY_VALUE = 10.0</tt>.
431 
432 The results files for submission are stored in the directory specified in the
433 <tt>OUTPUT_RESULT_DIR</tt> key in the <tt>[DIR.OUTPUT]</tt> section. Their names are of the form:
434 <tt>"results_branch_obs_type_data_type-img_no.txt"</tt>, for instance
435 <tt>"results_control_ground_constant-000.txt"</tt>. The "full" results file containing all
436 the fits is prefixed by <tt>full</tt>, like
437 <tt>"full_results_control_ground_constant-000.txt"</tt>.
438 
439 Lastly, @c gfit also produces for each processed image a files containing only the bad fits. These
440 files are stored in the directory pointed by the <tt>OUTPUT_ERROR_DIR</tt> key in the
441 <tt>[DIR.OUTPUT]</tt> section.

443 @subsection denoising Denoising
444 
445 The <tt>[DENOISING]</tt> section is intended to include any denoising scheme that may be applied
446 to each postage stamp.
447 
448 In this version, @c gfit only supports the DWT Wiener wavelet-based denoising algorithm from
449 Nurbaeva et al., 2011, whose settings can be set in the <tt>[DENOISING.DWTWIENER]</tt> section.
450 
451 @verbatim
452 [DENOISING]
453 
454 [DENOISING.DWTWIENER] # Denoising with dwtwiener (Nurbaeva et al., A&A, 2011)
455 
456 [DENOISING.DWTWIENER.GALAXY]
457 APPLY_DWTWIENER = False # if True apply dwtwiener to denoise the images
458 DECOMPOSITION_LEVEL = 2 # decomposition level (from l=1, default l=2)
459 CYCLE_SPINNING = False # if True, use cycle spinning
460 SIGMA_NOISE =-1 # noise standard deviation (estimated if set to -1)
461 DENOISING_STRENGTH = 0.6 # denoising strength fron 0 to 1

479 Additionally, if <tt>COLLECT_ERROR_DATA</tt> is set to @c True, @c gfit will collect additional data
480 besides fitted parameters, such as @c SExtractor data (from SExtrator catalogs) and statistics on
481 residuals.

485 key <tt>OUTPUT_PLOT_DIR</tt> in section <tt>[DIR.OUTPUT]</tt>. Otherwise, no plot will be made.

494 @subsection filtering Object Filtering
495 
496 This section provides a way to filter-out of "clean" unwanted objects, i.e.
497 galaxies or stars. A filtering criterion takes the form of a range of allowed values, like
498 <tt>[0, 600]</tt>. The value @c None denotes that there is no lower or upper bound. For instance,
499 <tt>[0, None]</tt> allows for values greater than zero, with no upper limit.
500 
501 Filtering takes place <em>before</em> fitting ("pre-filtering") or <em>after</em>
502 ("post-filtering").
503 - In this version, pre-filtering is applied on selected SExtractor catalog keywords. The
504 name of the criterion is the name of the SExtractor keyword prefixed by @c "SE_", like
505 @c SE_FLUX_RADIUS.
506 - Post-filtering can currently be done on the sum of the residuals between observed and built model,
507 normalized by the total number of pixels in the postage stamp used for fitting.
508 
509 The keyword @c FILTERING_POLICY specifies what should be done with the "filtered" objects. In this
510 version, objects are effectively "removed" from the final object catalog
511 (<tt>FILTERING_POLICY = remove</tt>). A future version may allow to ascribe a weighting factor
512 to filtered objects instead of removing them.
513 
514 Filtering can be enabled or disabled by specifying <tt>ENABLE_FILTERING=True</tt> or
515 <tt>ENABLE_FILTERING=False</tt>. If filtering is enabled, a catalog with un-filtered objects will
516 still be produced in order to better investigate the effect of filtering.
517 
518 @verbatim
519 [OBJECT_FILTERING]
520 
521 # --- Galaxy ---
522 [OBJECT_FILTERING.GALAXY]
523 
524 ENABLE_FILTERING = True # set to True to enable galaxy filtering
525 FILTERING_POLICY = remove # how to handle filtered data, among: [remove, weight]
526 
527 FILTER = {"SE_HLR" : [0, None],\ # allowed SExtractor FLUX_RADIUS range
528  "SE_FWHM_IMAGE" : [0, None],\ # allowed SExtractor FWHM_IMAGE range
529  "SE_FLUX" : [0, None],\ # allowed SExtractor FLUX_BEST range
530  "SE_MAG" : [None, None],\ # allowed SExtractor MAG_BEST range
531  "SE_SNR" : [0, None],\ # allowed SExtractor SNR (FLUX_BEST/FLUXERR_BEST)
532  "NORM_RESIDUALS" : [0, None] \ # allowed range of normalized sum of residuals
533  }
534 
535 # --- PSF ---
536 [OBJECT_FILTERING.PSF]
537 
538 ENABLE_FILTERING = False # set to True to enable PSF filtering
539 FILTERING_POLICY = remove # how to handle filtered data, among: [remove, weight]
540 
541 FILTER = {"SE_HLR" : [0, None],\ # allowed SExtractor FLUX_RADIUS range
542  "SE_FWHM_IMAGE" : [0, None],\ # allowed SExtractor FWHM_IMAGE range
543  "SE_FLUX" : [0, None],\ # allowed SExtractor FLUX_BEST range
544  "SE_MAG" : [None, None],\ # allowed SExtractor MAG_BEST range
545  "SE_SNR" : [0, None],\ # allowed SExtractor SNR (FLUX_BEST/FLUXERR_BEST)
546  "NORM_RESIDUALS" : [0, None] \ # allowed range of normalized sum of residuals
547  }

551 
552 The <tt>[LOGGING]</tt> section tells if logging is enabled, and which logging mode is used.
553 
554 @verbatim
555 [LOGGING]
556 MASTER_LOGGING_ENABLED = True # quiet mode if set to False
557 CREATE_WORKER_LOGGERS = per_job # among: {none, per_job, per_worker}
558 @endverbatim
559 
560 If the @c MASTER_LOGGING_ENABLED key value is set to False, no display output nor file logging will
561 be done.
562 
563 In the case <tt>MASTER_LOGGING_ENABLED=True</tt>, if the @c CREATE_WORKER_LOGGERS key value is set
564 to @c none, then no log file will be created for workers.
565 
566 If not @c none, the @c CREATE_WORKER_LOGGERS key specifies two logging modes: @c per_job and
567 @c per_worker. In the @c per_job mode, one log file will be created for each job processed (i.e.
568 each @c GREAT3 image). In the @c per_worker mode, one log file will be created for each worker, not
569 for each image, so the worker's log will trace the shear measurement output of all the images
570 processed by a particular worker.

583 EXTRACT_GALAXY_RESIDUALS = False # if True, extract galaxy residuals after fitting
584 EXTRACT_PSF_STAMPS = False # if True, write individual PSF stamps as .FITS files

599 
600 The number of postage stamps to fit within an image is set by the <tt>MAX_NB_STAMPS</tt>. A value
601 of @c -1 means that @e all postage stamps will be processed. otherwise, @c gfit will process the
602 @e first number of stamps specified in <tt>MAX_NB_STAMPS</tt>. The sorting order is y then x, so
603 @c gfit will proceed row after row in an image, processing each column of a given row.
604 
605 The @c EXTRACT_GALAXY_STAMPS, @c EXTRACT_CONVOLVED_GALAXY_STAMPS, @c EXTRACT_GALAXY_RESIDUALS,
606 @c EXTRACT_PSF_STAMPS boolean flags tell whether postage stamps should be extracted as .FITS
607 files for observed galaxies, convolved galaxies, galaxy residuals and observed PSFs, in this order.
608 The target directory is that specified as <tt>OUTPUT_RESULT_DIR</tt> in section
609 <tt>[DIR.OUTPUT]</tt>.
610

612 best-fitted convolved galaxy models inside postage stamps of size set by @c MOSAIC_STAMP_SIZE. The
613 coordinate of galaxies within the mosaic will be the same as in the source image and catalog. Failed
614 fits are indicated by a postage stamp with zero pixel values at the corresponding coordinates.<br>
615 Similarly, if @c CREATE_RESIDUALS_GALAXY_MOSAIC is set to @c True, a mosaic of the residuals
616 postage stamps will also be created, where residual postage stamps are the difference between
617 observed galaxies and model-convolved galaxies.<br>
618 Note that, in practice, most postage stamps will have an effective size smaller than the specified mosaic
619 stamp size (e.g. 44 instead of <tt>MOSAIC_STAMP_SIZE = 48) </tt>. Such stamps will be aligned on
620 the bottom-left corner of the mosaic stamp.
621 
622 The @c PLOT_GALAXY_STAMPS, @c PLOT_CONVOLVED_GALAXY_STAMPS, @c PLOT_GALAXY_RESIDUALS and
623 @c PLOT_PSF_STAMPS specify whether postage stamps should be plotted for observed galaxies,
624 convolved galaxies, galaxy residuals and observed PSFs, in this order.
625 The target directory is that specified as <tt>OUTPUT_PLOT_DIR</tt> in section
626 <tt>[DIR.OUTPUT]</tt>. If @c PLOT_ENABLE_3D is set to @c True, three-dimensional plots are also
627 drawn.
628

630 stored for later investigation in the directory pointed by <tt>OUTPUT_ERROR_DIR</tt> in section
631 <tt>[DIR.OUTPUT]</tt>.
632 
633 @c PROGRESS_COUNT specifies how often @c gfit should display or log output information. For
634 instance, a value of @c 10 will output every time @c 10 postage stamp images have been fitted.

647 @section multifit_config Multifit configuration
648 
649 @c gfit interacts with the Multifit module through a set of configuration files. There are
650 configuration files for fitting methods and for parametric models.
651 
652 @subsection model_config Galaxy Model configuration
653 
654 A Multifit configuration file is present for each supported galaxy model and bear the same name as
655 that of the model. Thus, there are @c 3 model configuration files in this version of @c gfit:
656 - <tt>scsersic.cfg</tt> for the single Sérsic component model
657 - <tt>gbdsersic.cfg</tt> for the bulge+disk model
658 - <tt>gcbdsersic.cfg</tt> for the "complex" bulge+disk model
659 
660 For each galaxy model, the list of model parameters is given, specifing the name of each parameter,
661 its guess value and its bounds (min and max alowed values).
662 
663 @subsection method_config Fitting Method configuration
664 
665 A Multifit configuration file is also present for each supported fitting method and bear the same
666 name as that of the method. Since @c 2 minimizers are supported in this version, there are also
667 @c 2 configuration files:
668 - <tt>scdmin.cfg</tt> for the @c SCDM minimizer
669 - <tt>scileastsq.cfg</tt> for the <tt>Levenberg-Marquardt</tt> non-linear least-squares
670  implementation available with @c SciPy.
671 
672 Sometimes, it is mecessary to provide fitting information that is common to both the model and
673 the method. This is the case for the @c SCDM minimizer, which uses such information for optimizing
674 the fitting process. For instance one can specify the initial step value to fit a given parameter.
675 Such extra fitting information can be kept in a file prefixed by the minimizer name and suffixed by
676 @c "_fitting". In the case of SCDM, known as @c scdmin by @c gfit, the fitting configuration file
677 is <tt>scdmin_fitting.cfg</tt>. It contains extra fitting information for each galaxy model.
