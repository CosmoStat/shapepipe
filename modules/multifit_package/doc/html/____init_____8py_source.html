6 @section Description
7 
8 The main objective of the MultiFit framework is to provide a common fitting interface to different
9 parametric models and fitting algorithms.
10 
11 To this end, MultiFit provides a common programming interface (API) to different minimizers and
12 parametric models.
13 This allows in particular to:
14 - Test how well different minimizers perform when fitting the same model. This is useful,
15  for instance to evaluate different implementations of the Levenberg-Marquardt non-linear least
16  squares algorithm (LVM) (e.g. levmar, http://users.ics.forth.gr/~lourakis/levmar/ or
17  scipy.leastsq, http://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.leastsq.html).
18 - Evaluate minimization algorithms other than LVM such as Coordinate Descent (CD),
19  http://en.wikipedia.org/wiki/Coordinate_descent.
20 - Keeping the same minimizer and the same data, select the model that give the most accurate
21  results. In the context of astronomy, one can for instance compare models based on
22  single-component Sersics and Bulge+Disk Sersics.
23 
24 This version is still in beta. Only the following minimizers and astronomical models are supported:
25 - Minimizers:
26  - @c scileastsq: based on the scipy Levenberg-Marquardt non-linear least available with the
27  Python scipy library
28  (http://docs.scipy.org/doc/scipy/reference/generated/scipy.optimize.leastsq.html))
29  - @c scdmin: an implementation of the Coordinate Descent algorithm, based on
30  SCDM (Simple Coordinate Descent Minimizer, Marc Gentile, 2013).
31 
32 - Parametric models:
33  - @c scsersic: an implementation of galaxy model base on a pure disk Sersic model
34  (http://en.wikipedia.org/wiki/Sersic_profile)
35  - @c gbdsersic: an implementation of galaxy model based on the combination of an exponential Sersic
36  profile (sersic index 1) to model the galaxy disk and a de Vaucouleur Sersic profile
37  (Sersic index 4) to model the galaxy bulge. Both the disk and the bulge share the
38  same centroid and the same elipticity. The underlying implementation is done using
39  @c GalSim v1.x, the "modular galaxy image simulation toolkit",
40  https://github.com/GalSim-developers/GalSim.
41  - @c gcbdsersic: an implementation of galaxy model identical to that of @c gbdsersic except the
42  disk Sersic index can also vary.
43  - @c profile2D: model of a generic two-dimensional profile, mostly for testing purpose.
44 
45 The software is designed so that additional models and fitting methods can be addded fairly quickly.
46 
47 @section code Source code
48 
49 The MultiFit code is organized as follows:
50 
51 - Configuration files:
52  - config/models: configuration for model: scsersic, gbdsersic, gcbdsersic, profile2D
53  - scsersic.cfg
54  - profile2D.cfg
55  - config/methods: configuration for methods: @c scdmin and @c scileastsq
56  - scileastsq.cfg
57  - scdmin.cfg
58  - profile2D.cfg
59  - config/fitting: extra fitting configuration that combine model and method information.
60  At present, only the scdmin method uses such configuration.
61  - scdmin_fitting: extra configuration for scdmin: @c scsersic, @c gbdsersic, @c gcbdsersic
62 
63 - Module files:
64  - modules/models: implementation of @c scsersic, @c gbdsersic, @c gcbdsersic, @c profile2D
65  - scsersic.py
66  - gbdsersic.py
67  - gcbdsersic.py
68  - profile2D.py
69  - modules/methods: implementation of @c scileastsq, @c scdmin
70  - scileastsq.py
71  - scdmin.py
72

74  - multifit.py
